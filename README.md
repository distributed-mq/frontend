# Message Queue

## 화면 설계
- 사용자는 '작업 1개 추가하기' 버튼과 '작업 10개 추가하기' 버튼을 누를 수 있다.
- 사용자는 MQ 에 대기 중인 작업의 수를 조회할 수 있다.

## 📌 화면 설계
### ✅ 좌측 (작업 추가)
- '작업 1개 추가하기' 버튼 (위에 위치, 파란색)
- '작업 10개 추가하기' 버튼 (아래 위치, 초록색)

### ✅ 가운데 (메시지 큐 상태)
- 현재 MQ에 쌓여 있는 작업 개수
- 개수는 파란색의 작은 박스로 표현됨.
- 최대 100개까지 저장할 수 있으며, 각 박스는 정사각형 형태이며, 옆에 있는 박스와 이격이 존재.
- 프로토 타입에서는 50개 고정된 상태로 표시 (API 연동 전)

### ✅ 우측 (Consumer 상태)
- Consumer 서버가 처리한 작업 개수
- 개수는 초록색의 작은 박스로 표현됨.
- 최대 100개까지 저장할 수 있으며, 각 박스는 정사각형 형태이며, 옆에 있는 박스와 이격이 존재.
- 프로토 타입에서는 50개 고정된 상태로 표시 (API 연동 전)

### ✅ 레이아웃 스타일
- 좌측 / 가운데 / 우측 세 구역으로 나뉨
- 얇은 실선으로 구분
